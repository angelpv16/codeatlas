## What Is a List?

A **list** is a container that holds multiple values in a specific order. Each value has a position — called an **index** — starting from 0. In Python, lists are the native equivalent of arrays found in other languages.

Think of it like a row of numbered boxes. Each box holds a value. You can access any box instantly by its number.

## Creating and Using Lists

```python
fruits = ["apple", "banana", "cherry"]
print(fruits[0])   # "apple"
print(fruits[-1])  # "cherry" (negative indexing)

fruits.append("mango")
fruits.remove("banana")
print(len(fruits)) # 3
```

Negative indexing lets you count from the end: `-1` is the last item, `-2` is the second to last, and so on.

## Key Operations

### Access by Index

```python
colors = ["red", "green", "blue", "yellow"]
print(colors[2])   # "blue"
```

### Slicing

```python
numbers = [10, 20, 30, 40, 50]
print(numbers[1:3])   # [20, 30]
print(numbers[:2])    # [10, 20]
print(numbers[3:])    # [40, 50]
```

### Iterating

```python
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)
```

### Append and Remove

```python
stack = [1, 2, 3]
stack.append(4)       # [1, 2, 3, 4]
stack.remove(2)       # [1, 3, 4]
```

## Time Complexity

Not all operations on a list are equally fast:

| Operation | Speed | Big O |
|---|---|---|
| Access by index | Instant | O(1) |
| Append to end | Fast | O(1) |
| Search by value | Slow for large lists | O(n) |
| Insert at start | Slow | O(n) |

**O(1)** means the time doesn't change regardless of list size. **O(n)** means it gets slower as the list grows.

## Common Mistake: IndexError

Trying to access an index that doesn't exist crashes your program:

```python
colors = ["red", "green", "blue"]
print(colors[5])  # IndexError: list index out of range
```

**Fix:** Always check the length first, or use negative indexing safely:

```python
colors = ["red", "green", "blue"]
if len(colors) > 5:
    print(colors[5])
else:
    print("Index out of range!")
```
