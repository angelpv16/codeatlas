## Two Structures, Opposite Rules

Stacks and queues are both built on top of lists, but they restrict **how** you add and remove items. This restriction is what makes them powerful — each one is perfect for a specific category of problems.

## Stack — Last In, First Out (LIFO)

Imagine a stack of plates. You always place a new plate on **top**, and you always take from the **top**. The last plate you added is the first one you remove.

```python
stack = []
stack.append("a")   # push
stack.append("b")
stack.append("c")
print(stack.pop())   # "c" — last in, first out
print(stack.pop())   # "b"
print(stack)         # ["a"]
```

### Real-World Stack Use Cases

- **Undo/Redo** — each action is pushed onto the stack; undo pops the last one
- **Browser back button** — visited pages are pushed; going back pops the last
- **Call stack** — when a function calls another function, the current one is pushed onto the stack

## Queue — First In, First Out (FIFO)

Imagine a line of people at a coffee shop. The first person in line is the first one served. New people join at the **back**, and people leave from the **front**.

```python
from collections import deque

queue = deque()
queue.append("first")
queue.append("second")
queue.append("third")
print(queue.popleft())  # "first" — first in, first out
print(queue.popleft())  # "second"
print(queue)            # deque(['third'])
```

### Real-World Queue Use Cases

- **Task scheduling** — OS processes tasks in the order they arrive
- **Print queues** — documents print in the order they were sent
- **BFS traversal** — explores graph nodes level by level using a queue

## Stack vs Queue at a Glance

| Property | Stack | Queue |
|---|---|---|
| Order | LIFO | FIFO |
| Insert | Top (push) | Back (enqueue) |
| Remove | Top (pop) | Front (dequeue) |
| Python | `list.append()` / `list.pop()` | `deque.append()` / `deque.popleft()` |

## Common Mistake: Using `list.pop(0)` for Queues

You might be tempted to use a regular list as a queue:

```python
# ❌ Slow — O(n) because all elements shift
queue = [1, 2, 3]
queue.pop(0)  # removes 1, but shifts 2 and 3

# ✅ Fast — O(1) with deque
from collections import deque
queue = deque([1, 2, 3])
queue.popleft()  # removes 1, no shifting needed
```

`list.pop(0)` is **O(n)** because every remaining element has to shift one position. `deque.popleft()` is **O(1)** — it just moves a pointer.

## What's Next?

Stacks and queues restrict access to make certain problems easy. But what if you need a structure where inserting and removing elements at **any position** is efficient? That's where **Linked Lists** come in.
